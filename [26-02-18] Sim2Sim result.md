# Blender to Genesis: Sim2Sim Transfer Analysis

## 1. 목표 (Goal)
**Blender 차량 애니메이션을 Genesis 물리 엔진에서 재현**하기 위해, Blender 경로에서 추출한 **가속도(a)**와 **곡률(κ)**를 Genesis에서도 동일하게 만들어내는 **throttle, steer 제어값**을 찾는 것.

> **Process:** Blender 경로 → a, κ 추출 → Genesis에서 같은 a, κ 재현 → throttle, steer 획득

---

## 2. 현재 방식: MPPI (Sampling 기반 MPC)

**매 프레임 실행 로직:**
1. **Sampling:** 200개의 랜덤 (throttle, steer) 시퀀스 생성 (Horizon=10) -> ``200개의 병렬환경과 sliding window size=10으로 실행`` 
2. **Rollout:** 각각 Genesis 물리 엔진에서 시뮬레이션 수행
3. **Cost Calculation:**  
  Cost = w_a|Δa| + w_κ|Δκ| + w_v|Δv| + w_θ|Δθ| + w_rate|Δu|
4. **Selection:** Cost가 가장 낮은 제어 입력을 가중평균으로 선택
5. **Update:** 첫 1 step만 적용 → 다음 프레임으로 이동

### 최적 파라미터 (Grid Search 결과)

| 파라미터 | 값 | 의미 | 비고 |
| :--- | :--- | :--- | :--- |
| **W_ACCEL** | `1` | a 매칭 | 가장 낮은 가중치 (희생됨) |
| **W_KAPPA** | `50` | κ 매칭 | 높은 우선순위 |
| **W_VEL** | `3` | 속도 보조 | |
| **W_HEADING** | `30` | 방향 보조 | 높은 우선순위 |
| **W_RATE** | `1` | 제어 Smoothness | 급격한 변화 방지 |
| **LAMBDA (λ)** | `0.01` | MPPI 온도 | 탐욕적 선택 유도 |

### LAMBDA (λ) 상세 설명
> λ는 MPPI에서 **"얼마나 탐욕적으로(Greedy) 선택할지"**를 결정하는 파라미터.

**가중평균 계산 식:**
`weight_i = exp(-cost_i / λ)`

* **λ = 0.01 (작음, 현재 설정) — 낮은 온도, 탐욕적:**
    * 시퀀스 #1: cost=0.5  → `exp(-50) ≈ 0` (무시)
    * 시퀀스 #2: cost=0.1  → `exp(-10) ≈ 0.00005` (거의 무시)
    * 시퀀스 #3: cost=0.01 → `exp(-1) ≈ 0.37` (**채택!**)
    * *결과: Cost가 가장 낮은 소수의 시퀀스만 선택됨*

* **λ = 1.0 (큼) — 높은 온도, 균등:**
    * 시퀀스 #1: cost=0.5 → `exp(-0.5) ≈ 0.61`
    * 시퀀스 #3: cost=0.01 → `exp(-0.01) ≈ 0.99`
    * *결과: 여러 시퀀스가 골고루 반영됨 (Averaging 효과)*

## 4. 원인 분석: 왜 이런 결과가 나왔는가?

**[핵심 모순]**
`a`, `κ`를 맞추려고 최적화했는데, 정작 `a`(79%), `κ`(42%) 오차는 크고, `v`(3.7%), `heading`(0.22°)이 잘 맞음.

### 원인 1: MPPI의 구조적 한계
* 200개 랜덤 샘플 중 "제일 좋은 것"을 선택하는 방식이므로, 정확한 최적해가 아닌 **근사해**임.
* `a`, `κ`는 순간적인 미분값이라 정밀한 제어가 필요한데 Sampling 방식으로는 한계가 있음.

### 원인 2: 가중치 불균형
* `W_ACCEL=1`(최저) vs `W_KAPPA=50`, `W_HEADING=30`
* MPPI가 `a`를 맞추는 것보다 `heading`/`κ`를 맞추는 것을 우선시함 → `a` 매칭이 희생됨.
* (Grid Search 결과 `W_ACCEL`을 높이면 전체 성능이 오히려 나빠짐 확인)

### 원인 3: 적분 효과 (Integral Effect) - **가장 결정적 이유**
* `a`의 순간 오차가 +/-로 진동 → 적분하면 상쇄됨 → **`v`는 잘 맞음**
* `κ`의 순간 오차도 진동 → 적분하면 상쇄됨 → **`heading`은 잘 맞음**
* **결론:** MPPI가 `a`, `κ` 자체를 맞춘 게 아니라, **"결과적으로 `v`, `heading`이 맞도록"** 학습한 것임.

> **MPPI의 실제 동작:**
> `a`, `κ` 직접 맞추기 (실패) → 대신 `v`, `heading`을 결과적으로 맞춤 (성공) → Position도 비교적 맞음

## 5. 남은 문제 (Remaining Issues)

**급커브 이후 속도 드리프트:**
* **현상:** 급커브 구간(Frame 280-320) 이후 Genesis 차량이 점점 빨라짐 (+0.8 m/s).
* **원인:** 가속도(`a`)가 정밀하게 맞지 않아서 미세한 과속 발생.
* **결과:** 이 속도 차이가 누적되어 최종 위치 오차 **2.7m**의 주 원인이 됨.

 ## 6. 일반화 문제
